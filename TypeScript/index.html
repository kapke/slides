Skąd on w ogóle się wziął?

  • Ogłoszony w 2012 roku w wersji 0.8 po 2 latach wewnętrzengo użytku w M$
  • Powstał z potrzeby tworzenia frontendów klasy enterprise
  • Twórca: Anders Hejlsberg (to ten od C#, Delphi i Turbo Pascala)

Założenia

  • Statyczne typowanie
  • Udostępnienie możliwości przyszłych wersji ECMAScript w obecnych dziś środowiskach uruchomieniowych
  • Dodawanie nowych funkcjonalności bez ograniczania istniejących

Możliwości

  • Wybór języka docelowego (ES3, ES5, ES2015)
  • Kompilacja do modułów (AMD, CommonJS, SystemJS, UMD) albo do jednego pliku
  • Dekoratory

Typowanie


class Sth<T> {
    private privateField:T;
    public publicField:T;

    public getSth ():T {
        return this.privateField;
    }
}

let sth:Sth<Foo> = new Sth(),
    foo = sth.getSth(); //foo is of type Foo!

ES 2015


import Foo from './Foo';

export default class Sth extends Foo {
    constructor (attrA) {
        super();
        this.fieldA = attrA;
    }

    someMethod* () {
        for(let i=0; ;i++) {
            yield i;
        }
    }

    anotherMethod () {
        return () => {
            return this.fieldA;
        };
    }
}
