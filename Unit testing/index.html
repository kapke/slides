Unit testing

What is purpose of automated testing at all?

Automation

There is no need to click through whole application

Documentation

Sometimes code is complicated to read, tests should be much easier

Give a confidence to programmer

With well-tested code it's easier and faster to make changes

What is a "unit"?

Object, class, function or what?

That depends on paradigm, even in single project it can be treated differently

So what is a "unit test"?

It's an automated test of specified unit performed in isolation from any other code

How to write test then?

1 logical assertion per test case

Each test case describes different use case of tested unit

Focus on behaviour, not internals

In a way that allows you to perform refactoring easily

Definition of refactoring is completely different story...

If possible - platform and framework independent

That means that unit tests are extremely fast, if they are not - probably they are not unit tests

SOLID, DRY, KISS, etc. - all these rules apply to tests as tests are code too

Bunch of tools in JS world

  • Karma - test runner
  • Jasmine - test framework
  • Mocha - test framework
  • Sinon.js - mocking utility
  • chai - assertion library
  • Jest - test framework with runner based on Jasmine
  • And more...
